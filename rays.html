copy_tls_property(dct=None)¶

Copies thread local data to a dictionary. rays.Applicaion object holds thread local data whose values are request specific. If you need to create child threads in a HTTP request, you must copy these data to the child threads.

>>> def make_worker(tls):
...   def run():
...     app.copy_tls_property(tls)
...     # operations...
...   return run
... @app.get("index")
... def index():
...   t = threading.Thread(target=make_worker(app.copy_tls_property()))
...   t.start()
...   t.join()

python index.py --server gunicorn --port 8000 -- -p index.pid -D

class rays.DatabaseExtension(app, connection, transaction='commit_on_success')¶

Database management extension.

Available configuration parameters:
 
connection

str, sqlite3 database file name.

transaction

str, transaction behavior.

  • commit_on_success : When a request starts, rays starts a transaction. If the response is produced without problems, rays commits transactions. Otherwise, rays rolls back transactions.
  • commit_auto
  • programmatic
